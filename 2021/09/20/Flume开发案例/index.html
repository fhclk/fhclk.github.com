Flume开发案例

发表于 2021-09-20

复制和多路复用

需求

使用 Flume-1 监控文件变动，Flume-1 将变动内容传递给 Flume-2，Flume-2 负责存储到 HDFS。同时 Flume-1 将变动内容传递给 Flume-3，Flume-3 负责输出到 Local FileSystem。

分析

实现

  1. 准备

    创建文件夹

  2. 创建flume-file-flume.conf

    配置1个接收日志文件的source和两个channel、两个sink，分别输送给flume-flume-hdfs和flume-flume-dir。

  3. 创建flume-flume-hdfs.conf

    配置上级 Flume 输出的 Source，输出是到 HDFS 的 Sink。

  4. 创建flume-flume-dir.conf

    配置上级 Flume 输出的 Source，输出是到本地目录的 Sink。

  5. 启动Flume

    分别启动对应的 flume 进程：flume-flume-dir，flume-flume-hdfs，flume-file-flume。

  6. 启动Hadoop和Hive

  7. 查看HDFS上数据

  8. 查看/root/data/flume3目录中的数据

负载均衡和故障转移

需求

使用 Flume1 监控一个端口，其 sink 组中的 sink 分别对接 Flume2 和 Flume3，采用FailoverSinkProcessor，实现故障转移的功能。

分析

实现

  1. 准备

    创建文件夹

  2. 创建flume-netcat-flume.conf

    配置 1 个 netcat source 和 1 个 channel、1 个 sink group（2 个 sink），分别输送给flume-flume-console1 和 flume-flume-console2。

  3. 创建flume-flume-console1.conf

    配置上级 Flume 输出的 Source，输出是到本地控制台。

  4. 创建flume-flume-console2.conf

    配置上级 Flume 输出的 Source，输出是到本地控制台。

  5. 执行flume

    分别开启对应配置文件： flume-flume-console2， flume-flume-console1， flumenetcat-flume。

  6. 使用netcat工具向本机的44444端口发送内容

  7. 查看Flume2及Flume3的控制台打印日志

  8. 将Flume2kill，观察Flume3的控制台打印情况

    注：使用 jps -ml 查看 Flume 进程。

聚合

需求

node1上的 Flume-1 监控文件/root/data/group.log，
node2上的 Flume-2 监控某一个端口的数据流，
Flume-1 与 Flume-2 将数据发送给 node3上的 Flume-3， Flume-3 将最终数据打印到控制台。

分析

实现

  1. 准备

    创建文件夹group3

    将flume同步到node2和node3节点

  2. 创建 flume1-logger-flume.conf

    配置 Source 用于监控 hive.log 文件，配置 Sink 输出数据到下一级 Flume。

    在 node1 上编辑配置文件 flume1-logger-flume.conf 。

  3. 创建 flume2-netcat-flume.conf

    配置 Source 监控端口 44444 数据流，配置 Sink 数据到下一级 Flume：

    在 hadoop103 上编辑配置文件 flume2-netcat-flume.conf

  4. 创建 flume3-flume-logger.conf

    配置 source 用于接收 flume1 与 flume2 发送过来的数据流，最终合并后 sink 到控制台。

    在 node3上编辑配置文件 flume3-flume-logger.conf 。

  5. 执行 flume

    分别开启对应配置文件： flume3-flume-logger.conf， flume2-netcat-flume.conf，flume1-logger-flume.conf。

  6. 在 node2上向/opt/module 目录下的 group.log 追加内容

  7. 在 node1上向 44444 端口发送数据

  8. 检查 node3上数据
